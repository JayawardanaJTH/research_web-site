Literature Survey

Currently, robots are used for different types of work, such as the manufacturing industry, healthcare, and the hotel industry. According to the current epidemic situation, the usage of robots was increased because of the need to reduce human interaction.

As a result, they have to walk around the workplace, because of that, they may have to climb staircases. The world has many types of robots. Here the selected robot is a humanoid. This proposal is concerned with how to detect the staircase, count steps, get dimensions of it, and move the robot on it by keeping body balance.

First, want to know what the objects are, then walk. The identified images from the image sensors will get as input. The technology stack that is used for image analysis is a method related to computer vision in deep learning. Other than that, while climbing the stairs robot needs to identify whether the staircase is over or not. Here we introduce a new concept: get the number of steps required to climb before climbing the stairs. It is related to how humans identify things by seeing and making decisions. The need to take the dimensions of the stairs is that when considering the stairs, they have their height, width, and range.

Therefore, it is imperative to calculate the dimensions of each staircase separately. Creating a balance system similar to human balance is a great advantage in robotics. To implement such a system, there is an upright pose controller to allow the robot to walk stably by preventing tilting of the robot during walking on uneven floor. In general, for us to do that we need to calculate the global inclination of the floor is a key factor. It can be measured with a 2-axis accelerator meter, and it is installed in the inertial sensor.

Research Gap

When talking about staircase detection, many researchers have done it in different ways. This system can develop by using a vision-based approach like a human. The human also sees objects through the eyes, images converted to a neuron signal. That signal sends to the vision area in the human brain. Then the brain can process that signal and give the vision, not only that it can detect the objects based on the previous information. As mentioned above human have eyes to see things likewise robot has many sensors, ultrasonic sensors, gyroscope sensors, tactile sensors, cameras, etc. In previous work, there are three main approaches to detect staircases. Those are sensor-based, vision-based, hybrid systems.

This research presents the theoretical analysis and implementation of autonomous staircase detection and stair climbing algorithms on a new rescue mobile robot. This is a system that is based on seniority. The main objectives are to locate the staircase during navigation and to develop an autonomous stair climbing algorithm that is fast, safe, and smooth. It is necessary to look for a certain number of levels. There are research groups who have developed autonomous robot registration and move using user instructions without any collision. Vision-based system was developed using the YOLO4 object determination algorithm.

Methodology

The robot which we selected is "Nao". It is developed by "Softbank" Robotics, and they do more developments on it. They have created a desktop application to develop their robots that are choreographed.So we have proposed the four main components.

Choregrapghe 2.8.6.23, Webots R2018a, Naoqi, and Python SDK(Software Development Kit) are the software we have used in this research. Choregraphe is a multi-platform desktop application that may run on Windows, Linux, and Mac computers. It is easy for beginners to create an NAO robot application due to its user-friendly interface. It is simple to create NAO robot applications without writing code using Choregraphe. Webots is an NAO robot simulation program. When designing and testing the animation of an NAO robot, the developer always tests in webots first, then checks whether everything is working properly using a real NAO robot to protect the real NAO robot’s motor. Python SDK is a programming kit that allows programmers to create NAO robot applications. It may be used to create Python modules that operate on the NAO robot or remotely. Furthermore, because the NAO robot’s CPU and memory are restricted, data may be saved to a remote PC using Python SDK. The NAO robot will use the remote PC to assist with picture processing and some complex computations.

Staircase detection is the main part of this research because other components depend on this component. There are different approaches to detect objects. In our scenario, we have used the vision-based method. This method is done by using the deep learning image classification technique. We have used VGG-16 and YOLO image processing models for our further development.

Gather distance is the second main part of this research because other dimensions calculations depend on distance. The Nao robot’s sonar sensor was used to calculate the distance. Using that sonar sensor, we can get the actual distance between the robot and the staircase.

Initially, we found out how to extract human motion from human walking, so we used a walking pattern generator to consider human characteristics (Walking Circle, Step length, Heel clearance, foot position) we had used here as a position-based approach. So, we have to implement a walking pattern generator and we have given here actual human data on walking and balancing the body, then when the robot starts to walk there is a feedback controller and it will compare the existing data in the generator and the current data of the robot. So, after that, if the current data is not in the possible range balance controller will balance the robot’s movement after the kinematic operation has been done. When the climbing staircase robot has to move upward its leg than the usual range, it will be detected by the feedback controller and the upper body will be controlled to bend forward, and we used the knee-ankle strategy to implement this climbing staircase in our proposed system.
